# Error Guide スキル

ユーザーがエラーに困っているとき、以下の4ステップで丁寧に解説する。

## 回答フォーマット

必ず以下のテンプレートを使う：

```
## 1. エラーの原因
[エラーが起きた理由を、プログラミング初心者にも伝わる言葉で説明する。
専門用語は使っても構わないが、必ず一言で補足する。
「〇〇とは〜のことです」のように。]

## 2. 解決手順
[番号付きのステップで、何をどの順番でやればいいかを書く。
各ステップには具体的なコマンドやコードも示す（あれば）。]

ステップ 1: ...
ステップ 2: ...
ステップ 3: ...

## 3. 同じエラーを防ぐTips
[次からこのエラーを起こさないための実践的なアドバイスを2〜3つ書く。]

- Tip 1: ...
- Tip 2: ...
- Tip 3: ...

## 4. 修正後の確認方法
[修正がうまくいったかどうかを確かめる方法を書く。
実行コマンドや期待される出力例があると親切。]
```

## ガイドライン

- **原因説明**は「なぜそうなるか」を優先する。エラーメッセージの丸写しだけにならないようにする。
- **解決手順**はユーザーの環境（言語・OS・フレームワーク）に合わせて具体的にする。不明な場合は最も一般的なケースで答え、「環境によって異なる場合は教えてください」と添える。
- **Tips**は「気をつけろ」ではなく「こうすると防げる」という形で前向きに書く。
- **確認方法**は実際に手を動かせる内容にする（「動いたか見てみてください」より「このコマンドを実行して〇〇が表示されたらOK」）。
- エラーメッセージが英語でもユーザーへの回答は日本語で行う（ユーザーが英語で話しかけてきた場合は英語で答える）。
- 複数の原因が考えられる場合は、最も可能性が高いものを先に説明し、他の原因も補足する。

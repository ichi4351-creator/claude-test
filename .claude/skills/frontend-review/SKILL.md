# frontend-review Skill

## トリガー
「フロントエンドレビューして」「HTMLをチェックして」「CSS/JSの観点を出して」
「レビュー観点を洗い出して」「デプロイ前チェックして」などのリクエストで使用する。

## 概要
HTML / CSS / JavaScript / アクセシビリティ / デプロイの5カテゴリに沿って
レビュー観点を表形式で洗い出す。**観点の提示のみ行い、実装は進めない。**

実装まで行う場合は `frontend-reviewer` SubAgent を使用すること。

---

## 出力フォーマット

各タスクごとに以下の表を出力する。

```
## タスクX：〈タスク名〉

| # | 観点 | チェック内容 | 深刻度 |
|---|---|---|---|
| H1 | HTML構造 | ... | 高/中/低 |
```

深刻度の基準：
- **高**：動作バグ・機能無効化・セキュリティ
- **中**：視覚的崩れ・パフォーマンス劣化
- **低**：コード品質・アクセシビリティ・保守性

---

## チェックリスト

### HTML
- [ ] セマンティックタグ使用（`<header>` `<section>` `<footer>` `<article>`）
- [ ] 全 `<img>` に `alt` テキスト設定（装飾画像は `alt=""` + `aria-hidden="true"`）
- [ ] インラインスタイル（`style=""`）がないこと → CSSクラスへ移動
- [ ] `role="dialog"` には `aria-labelledby` または `aria-label` を付与
- [ ] フォーカス管理が必要なモーダル・ライトボックスに `tabindex="-1"` 設定
- [ ] ギャラリーストリップ等の無限ループUIは複製枚数が正確か
  （`translateX(-50%)` アニメーションなら元セット数＝複製セット数）

### CSS
- [ ] `.active` 等のJS付与クラスに対応するCSSルールが存在するか
- [ ] インラインスタイルをCSSに吸収するユーティリティクラスが用意されているか
- [ ] `.sr-only`（スクリーンリーダー専用）クラスが定義されているか
- [ ] パラレックス要素は `will-change: transform` を設定しているか
- [ ] レスポンシブブレークポイントが全セクションをカバーしているか（1024px / 768px / 480px）
- [ ] アニメーション距離・タイミングがモバイルで過剰でないか

### JavaScript
- [ ] パラレックス計算がページ絶対座標ではなくセクション相対座標（`getBoundingClientRect()`）か
- [ ] `IntersectionObserver` の `threshold` と `rootMargin` が意図した発火タイミングか
- [ ] モーダル開閉時のフォーカス管理（開く→モーダル内ボタン、閉じる→元のトリガー）
- [ ] `aria-expanded` の値が文字列 `"true"/"false"` で正しくセットされているか
- [ ] スクロールイベントリスナーは `{ passive: true }` を指定しているか
- [ ] `querySelectorAll` の結果が空でもエラーにならないか（null チェック）

### アクセシビリティ（a11y）
- [ ] キーボードのみで全操作（ナビ・モーダル・閉じる）が完結するか
- [ ] `Escape` キーでモーダルを閉じられるか
- [ ] ループ用複製画像に `aria-hidden="true"` が付いているか
- [ ] スクリーンリーダー専用テキストは `.sr-only` クラスで視覚的に非表示にしているか
- [ ] カラーコントラスト比が WCAG AA 基準（4.5:1）を満たしているか

### GitHub Pages デプロイ
- [ ] `docs/` が GitHub Pages 配信元に設定されているか
- [ ] `.nojekyll` が `docs/` 直下に存在するか
- [ ] 画像パスが相対パスで記述されているか（絶対パス・`/`始まりは NG）
- [ ] `images/` 配下の全画像がコミットに含まれているか
- [ ] デプロイ後にコンソールエラーがないか確認したか
- [ ] 全セクション・ナビリンク・モーダルが本番 URL で正常動作するか

---

## 使用例

```
/frontend-review docs/index.html docs/style.css docs/main.js
```

観点を洗い出したあと、実装を進める場合は：

```
コードレビューを行いながらタスク処理を進めてください
```

と指示すると `frontend-reviewer` SubAgent が引き継いで修正まで行う。

# Summarize（ファイル要約スキル）

指定ファイルを読み込み、固定フォーマットで要約と改善案を出力する。
**要約本文（タイトル〜結論）の合計は1500字±10%（1350〜1650字）を目標とする。**

---

## Step 1: ファイルの存在確認

### ファイルパスが指定されていない場合
「要約するファイルのパスを教えてください」と尋ねる。

### ファイルが存在しない場合
```
❌ ファイルが見つかりません: `<指定パス>`
近いファイルを探しています...
```
Glob で類似ファイルを検索し、候補を提示して選択を求める。

### 複数ファイルが該当する場合（ワイルドカード等）
```
複数のファイルが見つかりました。要約するファイルを選んでください:
1. path/to/file1.py
2. path/to/file2.py
3. すべてを順番に要約する
```

---

## Step 2: 要約を出力する

ファイルを読み込んだら、以下のフォーマットで出力する。

```
## 📄 タイトル
[ファイルの内容を表す簡潔なタイトル（原題があればそれを使う）]

## 🔑 トピック
1. [最重要トピック：3〜4文で詳しく説明]
2. [第2トピック：3〜4文で詳しく説明]
3. [第3トピック：3〜4文で詳しく説明]

## ✅ 結論
[全体を通じた結論・要点を4〜5文でまとめる。このファイルが何のためにあるか・
読者が得られる知見・今後の展望などを含める]

---

## 💡 改善案
1. [改善点1: 具体的な問題と対応策を1〜2文で]
2. [改善点2: 具体的な問題と対応策を1〜2文で]
3. [改善点3: 具体的な問題と対応策を1〜2文で]
```

---

## 文字数の調整手順

1. 初稿を出力したら **タイトル〜結論** の文字数を自分でカウントする
2. 目標: **1350〜1650字**
3. 不足（1350字未満）→ 各トピックと結論に説明を追加する
4. 超過（1650字超）→ 冗長な言い回しを削り、簡潔にまとめる
5. 範囲内に収まったら改善案を追記して出力する

---

## ファイル種別ごとの要約指針

| ファイル種別 | タイトルの付け方 | トピックの選び方 |
|---|---|---|
| ドキュメント・MD | 文書の主題 | 主要な章・論点・主張 |
| Pythonスクリプト | モジュールの目的 | 主要な関数・クラス・処理フロー |
| 設定ファイル | 設定の対象システム | 重要な設定項目・影響範囲 |
| テストファイル | テスト対象と目的 | テストケースのグループ・カバレッジ |

## 改善案の観点（ファイル種別別）

| ファイル種別 | 改善案の観点 |
|---|---|
| ドキュメント・MD | 情報の過不足・読みやすさ・構成の論理性 |
| Pythonスクリプト | セキュリティ・可読性・エラー処理・型ヒント |
| 設定ファイル | セキュリティリスク・環境分離・コメント不足 |
| テストファイル | カバレッジの穴・エッジケース・テスト命名 |

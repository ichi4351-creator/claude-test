# Summarize（ファイル要約スキル）

指定されたファイルを読み込み、以下の固定フォーマットで要約を出力する。

## 出力フォーマット

必ずこの形式で出力する（日本語リクエストなら日本語、英語なら英語で）:

```
## 📄 タイトル
[ファイルの内容を表す簡潔なタイトル（原題があればそれを使う）]

## 🔑 トピック
1. [最重要トピック：1〜2文で説明]
2. [第2トピック：1〜2文で説明]
3. [第3トピック：1〜2文で説明]

## ✅ 結論
[全体を通じた結論・要点を2〜3文でまとめる]
```

## 手順

1. ファイルパスが指定されていれば Read で読み込む
2. パスが未指定なら「要約するファイルのパスを教えてください」と尋ねる
3. ファイルの種類（コード・文書・設定ファイル等）を判断して適切に要約する
4. トピックは内容から最も重要な3点を選ぶ（多くても3つ、少なくても3つ）
5. 結論は「このファイルが何のためにあるか・何を伝えているか」を中心にまとめる

## ファイル種別ごとの要約指針

| ファイル種別 | タイトルの付け方 | トピックの選び方 |
|---|---|---|
| ドキュメント・MD | 文書の主題 | 主要な章・論点 |
| Pythonスクリプト | モジュールの目的 | 主要な関数・クラス・処理フロー |
| 設定ファイル | 設定の対象システム | 重要な設定項目 |
| テストファイル | テスト対象と目的 | テストケースのグループ |

## 注意事項

- コードファイルの場合、コードそのものではなく「何をするコードか」を平易な言葉で説明する
- 長いファイルは全体を読んだうえで最重要な3点に絞る
- トピックは必ず3つ（多すぎず少なすぎず）
